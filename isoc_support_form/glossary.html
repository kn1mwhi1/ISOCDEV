AMC Billing Request
Request to run specific billing schedules from the AMC environment.

Bounce Daemon
To start and stop a daemon in the AMC environment.

Bypass
To skip-over a job/process/stream and continue with the next immediately available job/process available.

Cancel
Removes the job/process or stream from the current plan.

Dependencies
Jobs/Processes, files, and time restrictions that must be met in order for the job/process or stream to start running.

Hold
Changes the priority of a job/process or stream to prevent it from running at its regularly scheduled time.

Job Name
A command which ignores any un-met dependencies on a job/process stream. As long as priority and limits allow the job to run, manually releasing it should get the job/process or stream running immediately.

Kill
Stop a currently running job/process from executing. This will result in giving the job/process an abend state. A Killed job will be available for Re-Run or Bypass actions. A Killed job can also be Bypassed by confirming it to success.

Manually Release
A command which ignores any un-met dependencies on a job/process stream. As long as priority and limits allow the job to run, manually releasing it should get the job/process or stream running immediately.

Parameters
Any data that needs to be passed along to the job/process prior to it running. Parameters will affect how the job/process will run.

Re-Run
Refers to a job/process that is currently in 'Abend' or 'Successful' status that needs to be re-started or run, again.

Stream
A group of several jobs/processes linked together and usually dependent upon each other.

Submit Predefined
Refers to a job/process, or stream that is not available in the current plan. Also used for additional runs outside of the regularly scheduled instance. A-Hoc jobs will not be submitted with the usual Upstream and Downstream dependencies.

Successor
A downstream dependency of a job/process or stream. Downstream dependencies can be other jobs/processes or entire streams

Un-Hold
Returns a job/process or stream to its regular status. This may not cause the job/process to run immediately due to any unmet dependencies.
